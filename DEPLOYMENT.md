# Barnabas Dental — Deployment Guide

## Backend (Render)

1. Connect repo and use `render.yaml` (root directory: `backend` for web + cron services).
2. Set environment variables in Render dashboard:
   - `SECRET_KEY` — auto-generated by `render.yaml` for the web service; reuse the same value on cron jobs
   - `ALLOWED_HOSTS` — your Render API hostname
   - `DATABASE_URL` — Aiven PostgreSQL connection string
   - `CORS_ALLOWED_ORIGINS` — Vercel frontend URL
   - `FRONTEND_URL` — Vercel frontend URL
   - `EMAIL_HOST` — e.g. `smtp.gmail.com`
   - `EMAIL_HOST_USER` / `EMAIL_HOST_PASSWORD` — Gmail app password (SMTP)
   - `DEFAULT_FROM_EMAIL` — sender address
3. `DEBUG` is set to `False` in `render.yaml`.
4. Run `python manage.py createsuperuser` once after first deploy.

### Scheduled jobs (cron)

| Command | Schedule | Purpose |
|---------|----------|---------|
| `expire_pencil_bookings` | Every 15 min | Release expired 4-hour pencil holds |
| `send_appointment_reminders` | Daily 9:00 AM | 24-hour appointment reminders |
| `send_follow_up_reminders` | Daily 10:00 AM | Follow-up after completed visits (7 days) |

Cron services are defined in `render.yaml`. Ensure each cron job shares `DATABASE_URL` and `SECRET_KEY` with the web service. Reminder cron jobs also require `EMAIL_HOST`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD`, and `DEFAULT_FROM_EMAIL`.

## Frontend (Vercel)

1. Set root directory to `frontend`.
2. Set `VITE_API_URL` to `https://<your-api-host>/api`.
3. `vercel.json` handles SPA routing.

## Database (Aiven)

1. Enable automated backups in Aiven console.
2. Migrations run via Render `preDeployCommand`.

## SMTP security

SMTP credentials are **environment variables only** — not stored in the database or audit logs.

## Local development

```bash
# Backend
cd backend
cp .env.example .env
python manage.py migrate
python manage.py runserver

# Frontend
cd frontend
npm install
npm run dev
```

Manual pencil expiry (local): `python manage.py expire_pencil_bookings`
